# 统一响应处理实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为后端普通 JSON 接口建立统一响应结构，使业务异常返回 HTTP 200 + 自定义 code，非业务异常保留 HTTP 状态码。

**Architecture:** 通过 `AppError` 表达可预期异常，通过 `ErrorCodes` 统一错误码字符串，通过 `utils/response.js` 收敛 JSON 响应格式。全局 `errorHandler` 负责把异常转换为统一响应，路由层只做参数接收、服务调用和响应返回。

**Tech Stack:** Node.js >= 18、Express 4.x、CommonJS、node:test。

---

## 设计边界

- 只统一普通 JSON 接口响应，不强行包装文件下载流。
- `/api/ai/chat/stream` 保持 SSE 事件协议，不改为普通 JSON。
- 业务异常返回 HTTP 200，响应体 `success=false` 并携带业务 `code`。
- 参数错误、限流、未知系统异常保留 HTTP 状态码。
- 不引入第三方异常库，不做国际化，不做复杂错误码注册中心。
- 不执行 `git commit`、`git push`、建分支或改历史。

const ErrorCodes = require('../errors/errorCodes');

function ok(res, data = null, message = 'success') {
  return res.status(200).json({
    success: true,
    code: ErrorCodes.OK,
    message,
    data
  });
}

function fail(res, code, message, data = null, httpStatus = 200) {
  return res.status(httpStatus).json({
    success: false,
    code,
    message,
    data
  });
}

module.exports = {
  ok,
  fail
};
```

const AppError = require('../errors/AppError');
const ErrorCodes = require('../errors/errorCodes');
const response = require('../utils/response');

function errorHandler(err, req, res, next) {
  console.error('Error:', err);

  if (err instanceof AppError) {
    const httpStatus = err.isBusiness ? 200 : err.httpStatus;
    return response.fail(res, err.code, err.message, err.data, httpStatus);
  }

  return response.fail(res, ErrorCodes.INTERNAL_ERROR, 'Internal Server Error', null, 500);
}

module.exports = errorHandler;

```md
## 统一响应格式

普通 JSON 接口统一返回以下结构：

```json
{
  "success": true,
  "code": "OK",
  "message": "success",
  "data": {}
}
```

字段说明：

- `success`：请求是否达到业务成功状态。
- `code`：业务或错误码，成功时为 `OK`。
- `message`：面向调用方的简要说明。
- `data`：业务数据；失败时通常为 `null`。

业务可预期失败返回 HTTP 200，并通过 `success=false` 与 `code` 表达失败原因。参数错误、限流、未知系统异常等非业务异常保留对应 HTTP 状态码。

文件下载接口和 SSE 流式接口不强制包装为普通 JSON 响应。

## 错误码

| code | HTTP 状态码 | 类型 | 说明 |
| --- | --- | --- | --- |
| `OK` | 200 | 成功 | 请求成功 |
| `VALIDATION_ERROR` | 400 | 非业务异常 | 参数错误 |
| `PARSE_FAILED` | 200 | 业务异常 | 视频解析失败 |
| `VIDEO_UNAVAILABLE` | 200 | 业务异常 | 视频资源不可用 |
| `DOWNLOAD_RESOURCE_MISSING` | 200 | 业务异常 | 下载资源缺失 |
| `AI_CHAT_FAILED` | 200 | 业务异常 | AI 对话业务失败 |
| `RATE_LIMITED` | 429 | 非业务异常 | 请求过于频繁 |
| `INTERNAL_ERROR` | 500 | 非业务异常 | 未知系统异常 |
```

注意：如果 `API.md` 已有同类章节，合并而不是重复新增。
