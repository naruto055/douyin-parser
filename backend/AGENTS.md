# AGENTS.md

本文件为 `backend/` 局部协作说明，聚焦后端实现、测试与模块边界。

## 1. 模块职责

- `app.js`：应用入口与中间件、路由装配。
- `routes/`：HTTP 路由层，只做参数接收、调用服务、返回响应。
- `services/`：业务编排层，承载用例逻辑；避免掺杂大量 HTTP 细节。
- `utils/`：通用基础工具，保持薄而稳定，避免演变为业务杂物间。
- `ai/`：AI 运行时、工具注册、提示词、会话与模型集成。
- `middleware/`：横切逻辑，如限流、错误处理。
- `public/`：静态联调页面与前端调试脚本。
- `tests/`：Node.js 原生测试，按行为与模块覆盖核心能力。
- `docs/`：后端接口文档与局部设计说明。

## 2. 技术基线

- 运行时：`Node.js >= 18`
- 模块体系：当前以 `CommonJS` 为主
- Web 框架：`Express 4.x`
- AI 相关：`openai`、`@langchain/openai`、`langchain`、`zod`
- 解析链路：`puppeteer`、`puppeteer-cluster`、`puppeteer-extra`

## 3. 常用命令

在 `backend/` 目录执行：

```bash
pnpm test
pnpm start
pnpm dev
```

若仅修改后端代码，优先运行相关测试；若无法执行，需在结果中明确说明。

## 4. 改动落点约定

- 新增接口：优先放在 `routes/`，并将业务逻辑下沉到 `services/` 或 `ai/`。
- 新增业务编排：优先放在 `services/` 或 `ai/application/`，不要塞进路由层。
- 新增通用工具：仅在被多个模块复用时放入 `utils/`。
- 新增 AI 工具或运行时能力：优先放入 `ai/tools/`、`ai/runtime/`、`ai/infra/` 对应目录。
- 新增配置项：集中在 `config/` 与 `.env.example`，避免散落硬编码。

## 5. AI 模块约束

- 现有外部接口契约优先保持稳定，尤其是 `routes/ai.js` 暴露的普通响应与流式响应。
- 可以渐进调整 AI 内部实现，但不要无必要改动请求/响应结构。
- 提示词、工具注册、模型接入、会话管理应分层处理，避免单文件继续膨胀。
- 若同时存在旧实现与新运行时，优先做兼容式迁移，不做一次性大爆炸重写。

## 6. 代码风格要求

- 保持函数职责单一，避免“超大服务类”继续扩大。
- 倾向小而清晰的辅助函数，不引入没有收益的抽象层。
- 复用既有错误处理与响应模式，避免同类逻辑多套写法。
- 注释语言保持简体中文，仅在复杂逻辑前补充必要说明。

## 7. 测试与文档同步

- 修改路由、服务、AI 运行时或工具行为时，优先补充或更新对应测试。
- 接口契约变化时，同步更新 `backend/docs/API.md`。
- 若变更影响联调页面行为，检查 `public/` 下相关脚本是否需要同步。

## 8. 明确不要做的事

- 不直接修改 `node_modules/`、`temp/` 中的运行时产物。
- 不在路由层堆积业务逻辑。
- 不将仅服务单一场景的逻辑过早抽成“通用框架”。
- 不因局部需求随意改动全局配置或 API 契约。
