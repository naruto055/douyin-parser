# 抖音音频解析工具 - 技术方案

## 一、项目概述

**项目名称**：抖音音频解析工具
**项目目标**：提供一个 Web 服务，用户输入抖音分享链接，系统解析并提供音频优先的下载功能，同时支持视频下载。

---

## 四、详细技术实现

### 4.1 浏览器池设计 (puppeteer-cluster)

**核心配置**：
```javascript
const { Cluster } = require('puppeteer-cluster');
const puppeteer = require('puppeteer-extra');
const StealthPlugin = require('puppeteer-extra-plugin-stealth');

puppeteer.use(StealthPlugin());

// 浏览器池配置
const clusterConfig = {
  concurrency: Cluster.CONCURRENCY_PAGE,  // 按页面并发
  maxConcurrency: 3,                        // 最大 3 个浏览器实例
  puppeteerOptions: {
    headless: 'new',
    args: [
      '--no-sandbox',
      '--disable-setuid-sandbox',
      '--disable-blink-features=AutomationControlled',
      '--disable-dev-shm-usage',
      '--disable-accelerated-2d-canvas',
      '--disable-gpu'
    ]
  },
  puppeteer,  // 使用增强版 puppeteer
  retryLimit: 2,  // 失败重试 2 次
  retryDelay: 1000  // 重试间隔 1 秒
};

// 任务队列
cluster.task(async ({ page, data: url }) => {
  // 设置 User-Agent
  await page.setUserAgent(
    'Mozilla/5.0 (Windows NT 10.0; Win64; x64) ' +
    'AppleWebKit/537.36 (KHTML, like Gecko) ' +
    'Chrome/120.0.0.0 Safari/537.36'
  );

  // 拦截网络请求获取 API 数据
  let apiData = null;
  page.on('response', async (response) => {
    if (response.url().includes('/aweme/v1/web/aweme/detail/')) {
      apiData = await response.json();
    }
  });

  // 访问页面
  await page.goto(url, { waitUntil: 'networkidle2', timeout: 30000 });

  // 等待数据获取
  if (!apiData) {
    await page.waitForTimeout(2000);
  }

  return apiData || parseFromPage(page);
});
```

**浏览器池优势**：
- ✅ 减少浏览器启动开销（每次启动需 2-3 秒）
- ✅ 内存占用可控（限制最大实例数）
- ✅ 自动重试机制
- ✅ 任务队列管理

### 4.2 解析流程设计

```javascript
// 解析主流程
async function parse(url) {
  // 1. 预处理链接
  const realUrl = await resolveShortUrl(url);

  // 2. 优先使用 Puppeteer 浏览器池
  try {
    const data = await cluster.execute(realUrl);
    return {
      source: 'puppeteer',
      ...extractVideoInfo(data)
    };
  } catch (error) {
    console.log('Puppeteer 解析失败，尝试第三方 API:', error.message);
  }

  // 3. 降级到第三方 API
  const thirdPartyApis = [
    'https://api.xinyew.cn/api/douyinjx',
    'https://apis.jxcxin.cn/api/douyin',
    'https://www.devtool.top/api/douyin/parse'
  ];

  for (const api of thirdPartyApis) {
    try {
      const data = await callThirdPartyApi(api, realUrl);
      return {
        source: 'third-party',
        api,
        ...data
      };
    } catch (e) {
      console.log(`${api} 调用失败:`, e.message);
      continue;
    }
  }

  throw new Error('所有解析方式均失败，请检查链接是否有效');
}
```

### 4.3 音频提取策略

**音频优先流程**：
```
视频信息
    │
    ├─→ 检查是否有独立音频流 (music.play_url)？
    │       ├─ 是 → 直接提供音频流下载 (最快)
    │       └─ 否 → 进入视频提取流程
    │
    └─→ 从视频流提取音频
            ├─ 用户请求音频时
            │   ├─ 下载视频流
            │   ├─ FFmpeg 提取音频 (MP3, 192kbps)
            │   └─ 返回音频文件
            │
            └─ 用户请求视频时
                └─ 直接提供视频流下载 (按需)
```

**FFmpeg 音频提取实现**：
```javascript
const ffmpeg = require('fluent-ffmpeg');
const ffmpegPath = require('ffmpeg-static');

ffmpeg.setFfmpegPath(ffmpegPath);

async function extractAudio(videoUrl, outputPath) {
  return new Promise((resolve, reject) => {
    ffmpeg(videoUrl)
      .noVideo()
      .audioCodec('libmp3lame')
      .audioBitrate('192k')
      .audioFrequency(44100)
      .audioChannels(2)
      .on('start', (cmd) => {
        console.log('FFmpeg 命令:', cmd);
      })
      .on('progress', (progress) => {
        console.log('处理进度:', progress.percent + '%');
      })
      .on('end', () => {
        console.log('音频提取完成');
        resolve(outputPath);
      })
      .on('error', (err) => {
        console.error('音频提取失败:', err);
        reject(err);
      })
      .save(outputPath);
  });
}
```

### 4.4 下载策略设计

**API 设计**：
```
POST /api/parse
请求：{ url: "抖音链接" }
响应：{
  success: true,
  data: {
    title: "视频标题",
    author: "作者昵称",
    cover: "封面图 URL",
    duration: 15,  // 视频时长(秒)
    audioReady: true,  // 是否有独立音频流
    audioUrl: "音频下载地址 (如有)",
    videoUrl: "无水印视频地址"
  }
}

GET /api/download?type=audio|video&url=...
响应：文件流 (直接下载)
```

**前端交互流程**：
```
1. 用户粘贴链接 → 点击解析
   ↓
2. 显示加载动画
   ↓
3. 解析成功，显示：
   - 视频封面预览
   - 标题、作者信息
   - [下载音频] 按钮 (优先显示，高亮)
   - [下载视频] 按钮 (次要，普通样式)
   ↓
4. 用户点击 [下载音频]
   - 如有独立音频流 → 直接下载
   - 如无 → 显示"正在处理中..." → 服务端转换 → 下载
   ↓
5. 用户点击 [下载视频] (按需)
   - 直接提供无水印视频下载
```

## 七、关键技术点总结

| 技术点 | 方案 | 理由 |
|--------|------|------|
| 浏览器管理 | puppeteer-cluster | 复用浏览器，降低资源消耗 |
| 反检测 | puppeteer-extra-plugin-stealth | 隐藏自动化特征 |
| 解析策略 | 主 Puppeteer + 备第三方 API | 高可用，故障自动降级 |
| 音频提取 | FFmpeg (服务端) | 兼容性好，用户体验佳 |
| 下载策略 | 音频优先，视频按需 | 节省带宽，突出核心功能 |
| 进程管理 | PM2 | 自动重启，日志管理 |
| 反向代理 | Nginx | 静态文件服务，负载均衡 |

## 八、风险与应对

| 风险 | 应对措施 |
|------|----------|
| 抖音升级导致 Puppeteer 解析失败 | 自动降级到第三方 API |
| 第三方 API 全部失效 | 提示用户稍后再试，记录日志告警 |
| 内存占用过高 | 限制浏览器池大小，定时清理临时文件 |
| FFmpeg 转换超时 | 设置合理超时，支持断点续传 |
| 服务器 IP 被封禁 | 使用代理池，限制请求频率 |
